# Архитектура runtime

Chompo использует двухуровневую модель имён, чтобы совместить скорость исполнения и расширяемость хоста.

## Pipeline

```text
source
  -> Lexer
  -> Parser / AST
  -> Resolver
  -> Interpreter
```

Resolver не исполняет код и не меняет синтаксис. Он один раз вычисляет лексический адрес каждого имени.

## Локальные переменные

Локальная ссылка хранит пару:

```text
(depth, slot)
```

- `depth` — сколько Environment пройти к родителю;
- `slot` — индекс значения в плотном массиве нужного Environment.

Во время исполнения локальное чтение не хеширует строку и не перебирает scope. Оно выполняет переход к ancestor и индексирует `vector<Value>`.

Такая модель работает для:

- параметров;
- вложенных блоков;
- shadowing;
- рекурсии;
- closures;
- функций, объявленных внутри блоков.

## Глобальные и native-имена

Глобальные значения остаются в реестре по `SymbolId`.

```cpp
interpreter.install_collection_builtins();
interpreter.install_io_builtins(io_manager);
interpreter.install_network_builtins(network_manager);
```

Новый native-модуль по-прежнему регистрирует функции строковыми именами. Для добавления встроенной функции не требуется изменять resolver, AST или формат локальных slots.

## Совместимый fallback

`Environment` сохраняет динамический map-based путь для host-кода и будущих динамических модулей. Код, прошедший Resolver, использует быстрые slots; неразмеченные host-обращения могут использовать прежний поиск.

Это позволяет позже добавить:

- модули и импорт;
- классы и методы;
- plugin API;
- REPL с динамическими объявлениями;
- bytecode backend;

без возврата локальных переменных к строковым lookup.

## Closures

Closure хранит `shared_ptr` на лексическое Environment. Лексический адрес остаётся корректным после завершения внешней функции, потому что цепочка Environment продолжает существовать.

Пул function frames переиспользует только Environment, которое не удерживается closure. Захваченные frames никогда не очищаются или не выдаются другому вызову.

## Семантика

Оптимизация не изменяет:

- порядок вычисления выражений;
- правила shadowing;
- ссылочную семантику массивов;
- поведение closures;
- runtime-ошибки неопределённых и повторно объявленных переменных;
- API встроенных функций.
